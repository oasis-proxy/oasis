# 监控与分析

Oasis 提供了网络流量的透明展示，详细显示被代理或直接连接的请求及其策略路由逻辑。

## 1. 实时请求监控页面

请求监控是一个全屏分析工具，提供 Oasis 处理的所有浏览器请求的实时提要。

### 日志详情

监控器为每个请求显示以下信息：

- **方法与状态**: (例如：GET 200, POST 404)。
- **域名与 IP**: 目标主机名和实际请求服务的 IP 地址（如果使用了代理，可能是代理服务器地址）。
- **时长**: 连接所花费的总时间。
- **代理节点**: 所使用的特定代理服务器或代理组。
- **规则来源**: 识别触发路由的特定规则（通配符、正则表达式或规则集）。

### 高级过滤与管理

- **标签页过滤**: 筛选特定浏览器标签页的流量。
- **搜索查询**: 按域名、IP 或代理名称进行实时日志过滤。
- **状态过滤**: 筛选失败的请求（错误/超时），用于调试连接问题。
- **一键清理**: 支持清理当前视图中的所有记录或仅清理特定标签页的历史。

---

## 2. 一键批量添加规则 (域名收纳)

监控器不仅用于分析，更是高效配置的入口。对于请求密集的复杂网站，可以使用“添加标签页域名”功能。

### 工作流

1. 在左侧侧边栏中选择一个特定的 **浏览器标签页**。
2. 点击顶部的 **添加标签页域名 (Add Tab Domains)** 按钮。
3. 系统将自动整理该标签页产生的所有唯一域名。

### 智能合并

> [!TIP]
> 批量添加利用了 Oasis 的“智能合并 (Smart Merge)”引擎，会自动建议将属于同一父域名的多个子域名收纳为通配符模式（例如：将 `a.com` 和 `b.a.com` 整理为 `*.a.com`），从而保持策略集的简洁。

---

## 3. 连接监控 (Popup)

**Popup** 提供“迷你监控器”，支持在不跳转页面的情况下进行快速状态检查。

- **活动跟踪**: 列出活动标签页请求的所有域名及实际请求的 IP 地址。(以域名去重合并)
- **协议警告**: 指示站点请求报错信息。

_注：实时追踪功能可在 [扩展配置](EXTENSION_CONFIG.md) 中开启。_
