# 扩展配置

全局行为自定义选项，支持根据特定浏览要求调整 Oasis。

## 1. UI 自定义

Oasis 界面现代且响应迅速，支持多种环境外观。

### 主题引擎

- **浅色模式 (Light Mode)**: 高对比度的白色界面。
- **深色模式 (Dark Mode)**: 灰色调的深色界面。
- **系统 (自动)**: 自动匹配操作系统的外观。

### 右键菜单

- **显示右键菜单**: 控制是否在浏览器右键菜单中显示 Oasis 相关选项。（参考：[交互功能](INTERACTIVE.md)）

---

## 2. 浏览器行为与路由

管理 Oasis 与浏览器标签页或网络请求之间的交互逻辑。

### 切换后刷新

> [!TIP]
> “切换后刷新”支持切换代理时自动重新加载当前的标签页，确保新的代理策略立即生效。

### 连接监控

- **启用监控**: 开启对网络请求的域名、IP 以及连接时长的实时追踪。（参考：[监控与分析](MONITORING.md)）

### IP 标签 (IP Tags)

> [!TIP]
> IP 标签支持为经常访问的服务器节点分配描述性标签。

- 可为任何 IP 地址配置自定义标签（例如：“主 API”、“CDN 节点 1”）。
- 标签出现在全屏监控器和迷你视图（popup）中，便于快速识别服务器。

### 拒绝地址配置

- 被“拒绝 (Reject)”规则或策略将在内部拦截，服务端无法收到该请求（实现方式：将请求重定向到一个不存在代理服务的端口）。
- 默认使用 127.0.0.1:65535，如本地有其他服务使用了该端口，可变更为一个不存在服务的端口。

### 全局规则优先级权重

- 允许调整规则大类的优先次序。
- 默认优先级为：**拒绝 > 临时 > 正常**。

---

## 3. 维护与更新

远程资源（PAC 脚本和规则集）的自动化更新通过维护周期进行管理。

### 全局更新频率

- 支持配置所有订阅的更新检查间隔（15 分钟到 24 小时）。

### 缓存管理

- 显示远程资源的“上次更新”时间戳。
- 支持手动清除本地缓存以进行维护。
